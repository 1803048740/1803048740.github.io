# SQL优化

- 避免在索引列上使用计算和函数

- 避免在索引列中发生隐式转换

- 避免使用OR

- 避免使用

- 避免使用LIKE

- 表连接代替子查询

- EXISTS代替IN

- 用<和>代替<>

- 能放在WHERE后面的就不要放在HAVING后面





> 索引失效
>
> 1.索引本身失效
>
> 2.隐式转换导致索引失效
>
> 3.查询的数据量为一个表的大部分数据
>
> 4.对小表查询
>
> 5.B-TREE索引 IS NULL 不走索引  IS NOT NULL 走索引
>
